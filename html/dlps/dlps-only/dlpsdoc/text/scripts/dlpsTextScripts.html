flipent Processing Scripts
Description Converts (flips) character entity references from hexadecimal (like &#xA9;) to mnemonic/named (like &copy;)
User interface Command line
Usage flipent [options] filenames
Options
option usage description
q (quiet) -q Do not show summary of replacements; only show error messages
Input
argument description
filenames (required) TEI XML file(s)
Output Copies each input file to [filename].bak, then overwrites original input file; also sends a summary of replacements (and of references that couldn't be replaced) to standard output
Remarks The script uses a list of character entities built from the standard SGML (ISO 8879) character-entity sets (iso-lat1.ent, etc. plus uva-supp.ent). If the input file contains a hexadecimal entity reference that is not on that list, the hex reference simply does not get replaced.

After making its replacements, the script runs the declare_charents script to declare/insert any undeclared character-entity-set files.

header_date_values Processing Scripts
Description Corrects common problems with value attributes on <date> elements in the TEI header
User interface Command line
Usage header_date_values filenames
Options
option usage description
C (Cataloging mode) -C Output will include an "ATTN Cataloging..." XML comment for any date value in the TEI header that is not a 4-digit year
Input
argument description
filenames (required) TEI XML file(s)
Output Copies each input file to [filename].bak, then overwrites original input file
Remarks
  • The problems corrected by this script are common in TEI headers derived from MARC records. Such problems result in error messages from the qa_dates script (or qa_super -d).
  • The script only changes values in the TEI header; no other part of the file is affected.
  • The script does not change the content of the <date> element (only the attribute values).
  • The script fixes the following problems:
    • if value is enclosed by square brackets, removes them
    • if value ends with "?", removes ? and adds certainty="ca"
    • if value starts with "ca." or "c.", removes it and adds certainty="ca"
    • if value has form "1584 or 5", removes " or #" and adds certainty="ca"
    • if value starts with "after", removes it and adds certainty="after"
    • if value starts with "before", removes it and adds certainty="before"
    • if value starts with "b.", removes b. and adds type="birth"
    • if value has form "1847-" and occurs inside <author>, removes hyphen and adds type="birth"
    • if value starts with "d.", removes d. and adds type="death"
    • if value starts with "fl.", removes fl. and adds type="fl"
    • if value is two years separated by a hyphen, converts to <dateRange>
Examples header_date_values *.xml
XML Before and After 
<date n="534|c" value="[1859?]">[1859?]</date>
<date n="534|c" value="1859" certainty="ca">[1859?]</date>
<date n="100|d" value="b. 1855">b. 1855</date>
<date n="100|d" value="1855" type="birth">b. 1855</date>
<date n="100|d" value="1796?-1850">1796?-1850</date>
<dateRange n="100|d" from="1796" to="1850" exact="to">1796?-1850</dateRange>
<date n="100|d" value="fl. 1890-1900">fl. 1890-1900</date>
<dateRange n="100|d" from="1890" to="1900" type="fl">fl. 1890-1900</dateRange>

postkb Processing Scripts
Description Runs a series of post-keyboarding scripts on TEI XML files
User interface Command line
Usage postkb [options] filenames
Options
option usage description
f (foreign) -f Indicates a foreign-language (non-English) text; skip de-hyphenate script [same effect as -h option; retained for backward compatibility]
h (keep hyphenation) -h Skip dehyphenate script

Input
argument description
filenames (required) TEI XML file(s)
Output Copies each input file to [filename].bak, then overwrites original input file
Remarks The scripts run by postkb are:
  • dos2unix — Converts to Unix line endings
  • last_line — Adds a newline at end of file if needed
  • UNK2unknown — Converts vendor [UNK] indicators to &unknown;
  • replace_regexp_chars — Replaces characters ( ) [ ] with character entities
  • flipent — Flips character entity references from hexadecimal (like &#xA9;) to mnemonic/named (like &copy;)
  • multiline_tags — Fixes start-tags broken across a line break
  • replace_xml_decl — Adds or replaces XML declaration
  • dlps_properties — Adds <?dlps ...?> properties
  • replace_system_paths — Replaces SYSTEM paths (sets locations of DTD files)
  • vendor2tei — Converts vendor markup to standard TEI markup
  • div_ids — Adds id attribute on <div#> tags
  • dehyphenate — Collapses end-of-line hyphenation
  • declare_charents — Declares character entity sets as needed

  • size is the value to be used by the mogrify command's -scale option; format is [width]x[height], where [width] and [height] are integers indicating maximum possible pixels
  • move is a yes/no value indicating whether or not to move the images to the destination directory after they're created in the source directory
  • verbose is a yes/no value indicating whether or not to display info messages (as well as warning and error messages, which are always displayed)
  • IDs to be processed must be specified one per line

With the exception of the IDs to process, the settings are all optional, since they all have defaults; the default values are indicated in the example config format above.
Interaction with user Displays configuration settings and list of IDs that will be processed. User indicates yes (use these settings), or no (stop the script).

rimage.pl Administrative/Utility Scripts
Description Creates archival discs using the Rimage DVD burner
User interface Command line
Usage perl C:\DLPS\rimage\rimage.pl [options] paths
Options
option usage description default
b (base ID) -b base ID Base ID to use when determining disc ID Defaults to the current date in YYMMDD format
c (copies) -c number Number of copies to make of each disc 2 (The default is 2 because it is standard DLPS practice to burn two copies of each disc and store them in separate locations.)
k (keep) -k Keep disc image (.img) files after producing the discs Default behavior is to delete each disc image after burning from that image is complete
l (label) -l label name Informal name of label template: allowed values are "DLPS", "FKL", "TBRC", "SPEC" and "THDL" "DLPS"
m (media) -m media name Type of disc media: allowed values are "DVD" and "CD" "DVD" (In practice, DLPS does not use CDs for archival storage.)
t (test) -t Report how many discs will be needed and what their IDs will be, but do not create any editlists, log files, disc images, etc. and do not actually burn any discs
v (verbose) -v Show status messages (in addition to warnings and errors)
I (imaging only) -I Only create disc images; do not produce discs Default behavior is to create disc images and produce discs from those images
P (production only) -P Only produce discs from existing disc images; do not create disc images Default behavior is to create disc images and produce discs from those images
Input
argument description
paths (required) Path(s) to one or more directories containing the files to be burned. If using the -P option, the directory (only one) containing the disc images to burn.
Output The main output is a set of burned discs.

The script performs the following tasks:

  • recursively processes each file and subdirectory in the specified input directories
  • calculates how many files will fit on each disc, and determines how many discs will be required
  • creates a "merge" file for each disc — a comma-separated file with summary info that gets merged into a label template for the disc label (under C:\Rimage\Merge)
  • creates an "editlist" file for each disc — a list of files to be burned to the disc (under C:\Rimage\Editlist)
  • creates a log file for each disc that records important information about the disc, including a list of the files contained on the disc (under C:\Rimage\Logs)
  • creates a configuration file for the accompanying Java application (C:\Rimage\DLPSRimageConfig.xml)
  • runs a Java command-line application that handles all communication with the Rimage system
Remarks It is the accompanying Java application that actually controls the Rimage system.
  • The Java app tells the Rimage to create a disc image file for each disc; as each disc image is completed, that image is burned (as many times as indicated by the -c (copies) option).
Examples perl C:\DLPS\rimage\rimage.pl E:\myFiles
Burn contents of E:\myFiles

perl C:\DLPS\rimage\rimage.pl -t E:\myFiles
Report how many discs will be needed (and what their IDs will be) to burn the contents of E:\myFiles, but do not produce any discs

perl C:\DLPS\rimage\rimage.pl E:\project1 E:\project2 E:\project3
Burn discs for three different projects. Within each project, each disc will contain as many files as possible, but each project will begin on a new disc.

perl C:\DLPS\rimage\rimage.pl -l TBRC -b TIBXHD03 -c 1 V:\TBRC3_eBooks
Burn contents of V:\TBRC3_eBooks; use the "TBRC" custom disc label (rather than the DLPS default label); use "TIBXHD03" as the base ID for each disc (rather than a 6-digit date); burn only one copy of each disc (rather than two)
Language Perl for Win32
Location C:\DLPS\rimage
