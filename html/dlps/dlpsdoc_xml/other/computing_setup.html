Computing Setup for New Staff Members

This document describes the steps needed to set up the computing environment for new staff members in Digital Library Production Services (DLPS), University of Virginia Library. It is intended for use by internal DLPS staff.

Unix Environment Setup

Unix Accounts
  • Request a Unix account for each new staff member. Library IT Systems (LITS) handles the creation of accounts; they have a web form at http://staff.lib.virginia.edu/systems/libsys/help-request.php for submitting such requests.
    • Provide the new user's UVa computing ID (email ID)
    • Request that dlpswg (DLPS working group) should be the new user's default Unix group
    • Request that the new user get added to the htaccess permissions for web-based tools/documentation under /www/doc/dlps/dlps-only on pogo.lib
    • If (and only if) this staff member needs to run the dlps2ReadyRepo script, then request that this user get added to the Unix group repo
      IMPORTANT

      Adding a user to repo grants that user write permissions (including delete permissions) to the entire repository hand-off point under /cenrepo/ReadyRepo. Only request this if you're certain it is needed. If in doubt, don't add the user to repo initially; this request can always be made later.

  • When the account is created, LITS will notify you that the account is ready and tell you how to proceed. Normally the first step is to have the new user log onto viva.lib and change the default password.
  • At this point the new user should be able to log into the Library's Unix servers, including the Linux server we use for our digital production work, pogo.lib (also known as alioth.lib).

Environment Variables

To work effectively from the Unix command line, each user needs to set a few environment variables:

  • Have the new user log in using his/her username and password. After login, you should be in the user's home directory.
  • Using Jove (or some other text editor), open the user's initialization file. Normally you'll want to edit the .variables.ksh file, since ksh is the default shell (command interpreter).
  • Add the following lines at the end of the initialization file: 
    # add DLPS bin directory to path
PATH=/shares/admin/bin:/shares/admin/bin/text:/shares/admin/bin/image:${PATH}

# set umask to 002 to grant group write permission on new
# files/directories; this sets default permissions to 664 (rw-rw-r--)
# for new files, 775 (rwxrwxr-x) for new directories
umask 002
  • Save and close the initialization file.
  • Log off and then log back in (necessary for the new setup to take effect).
  • To test the new settings, enter: 
    echo $PATH
    You should see a colon-delimited list of paths, starting with the paths you added to the initialization file in the preceding step.
