Setup for Rimage Systems

This document describes the setup requirements for the Rimage DVD recorders at Digital Library Production Services (DLPS), University of Virginia Library. It is intended for use by internal DLPS staff.

Note: This documentation pertains to version 6.5 of the Rimage OfficeNet software.

Last modified: 11/16/2006

Setup Requirements

DLPS has two Rimage systems for burning and reading DVDs — the Desktop 1500 and the Rimage 2000i. Both systems use the same software and have the same setup requirements.

  • Rimage software — The Rimage software is called OfficeNet and is available on CD from the Rimage company. Ask your LSP to install the Rimage software on the Rimage PC.
    NOTE

    Because DLPS has a maintenance contract with Rimage, Rimage will periodically send us a software upgrade on CD. Installation is straightforward. Install the upgrade on both of the Rimage PCs in DLPS.

    Rimage also makes service packs and minor updates available on the Rimage web site. As of 11/16/2006, both systems have been updated with Rimage Software Suite 6.5 Service Pack 4.

  • Perl — The standard port of Perl for Windows is called ActivePerl and is available here (right-click and select Save Target As...), or as a free download from

    http://www.activestate.com/Products/ActivePerl/

    Ask your LSP to install Perl. To test the installation, open a command window and enter: perl -v
  • Java runtime — Under the hood, rimage.pl actually runs a Java program to handle communication with the Rimage system, so the Java Runtime Environment (JRE) must be present. The JRE is available here, or as a free download from

    http://java.com/

    Ask your LSP to install Java. To test the installation, open a command window and enter: java -version
  • Supporting Java classes — Additional Java software required for communicating with the Rimage system comes from several sources:
    • Rimage classes — The Rimage .jar files are installed with the other Rimage software (see C:\Program Files\Rimage\Messaging\Lib).
    • JAXP (Java API for XML Processing) — Facilitates reading XML documents via Java. Available here, or from http://java.sun.com/xml/jaxp/index.jsp
    • JMS (Java Message Service) — A standard for sending/receiving messages between software components. Available here, or from http://java.sun.com/products/jms/
    • JDOM (Java Document Object Model) — Facilitates building XML documents in Java. Available here, or from http://jdom.org/
  • Other required files — Several other files, which are available for download here, are also required:

    The following files must be in the C:\Rimage directory on the Rimage PC:

    • DLPSRimageConfig.dtd
    • DLPSRimageConfig.xml
    • log.properties

    The following files must be in the C:\DLPS\rimage directory on the Rimage PC:

    • rimage.pl
    • dlpsRimage.jar

    This file must be in the C:\Rimage\Labels directory on the Rimage PC:

    • DEFAULT.btw

  • CLASSPATH — The .jar files for the various Java technologies need to be included in the CLASSPATH environment variable. To add or edit environment variables, go to Start → Control Panel → System → Advanced tab → Environment Variables.
    TIP

    In Windows, there are two types of environment variables: user and system. For use with the Rimage, the CLASSPATH variable needs to be set system-wide. To add or edit system variables, you must be logged into Windows as a user with Administrator privileges.

    For convenience, set up some new environment variables to specify the directories where the .jar files reside. For example:

    Variable Value
    RIMAGE_JARS C:\Program Files\Rimage\Messaging\Lib
    JAXP_HOME C:\Java\jaxp-1.2_01
    JMS_HOME C:\Java\jms1.1\lib

    Then copy and paste the following into the CLASSPATH environment variable:

    %RIMAGE_JARS%\activation.jar;%RIMAGE_JARS%\AdminApi.jar;%RIMAGE_JARS%\ClientApi.jar;%RIMAGE_JARS%\CommonApp.jar;%RIMAGE_JARS%\log4j-1.2.7.jar;%RIMAGE_JARS%\MsgApi.jar;%RIMAGE_JARS%\sonic_Client.jar;%RIMAGE_JARS%\sonic_Selector.jar;%RIMAGE_JARS%\tools.jar;%JAXP_HOME%\dom.jar;%JAXP_HOME%\jaxp-api.jar;%JAXP_HOME%\sax.jar;%JAXP_HOME%\xalan.jar;%JAXP_HOME%\xercesImpl.jar;%JAXP_HOME%\xsltc.jar;%JMS_HOME%\javax.jms.jar;%JMS_HOME%\jms.jar;C:\Java\jdom-1.0\build\jdom.jar;C:\DLPS\rimage\dlpsRimage.jar

    which breaks down as follows:

    • Rimage classes —
      • %RIMAGE_JARS%\activation.jar
      • %RIMAGE_JARS%\AdminApi.jar
      • %RIMAGE_JARS%\ClientApi.jar
      • %RIMAGE_JARS%\CommonApp.jar
      • %RIMAGE_JARS%\log4j-1.2.7.jar
      • %RIMAGE_JARS%\MsgApi.jar
      • %RIMAGE_JARS%\sonic_Client.jar
      • %RIMAGE_JARS%\sonic_Selector.jar
      • %RIMAGE_JARS%\tools.jar
    • JAXP classes —
      • %JAXP_HOME%\dom.jar
      • %JAXP_HOME%\jaxp-api.jar
      • %JAXP_HOME%\sax.jar
      • %JAXP_HOME%\xalan.jar
      • %JAXP_HOME%\xercesImpl.jar
      • %JAXP_HOME%\xsltc.jar
    • JMS classes —
      • %JMS_HOME%\javax.jms.jar
      • %JMS_HOME%\jms.jar
    • JDOM classes —
      • C:\Java\jdom-1.0\build\jdom.jar
    • the DLPS Rimage Java application —
      • C:\DLPS\rimage\dlpsRimage.jar
