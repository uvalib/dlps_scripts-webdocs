Non-transcription Workflow

This document describes the workflow for creating non-transcription texts at Digital Library Production Services (DLPS), University of Virginia Library. It is intended for use by internal DLPS staff.

A "non-transcription text" is a TEI resource that has associated page images and a complete TEI header, but no full-text transcription.

Create TEI XML Files

The script that creates non-transcription TEI XML files is make_tei_nontrans. Because it uses predefined input and output directories, the script can be run from any location.

  • Determine page image type: if bitonal, no special options are needed; if the page images are color, use the -c option, as shown below
  • Run the script, indicating the DLPS IDs for which to create TEI XML files: 
    make_tei_nontrans -c b001234567 b007654321
  • For each DLPS ID specified, the script creates a TEI XML file in /shares/text/04postkb/70being_fullheaders_added

Continue Processing
  • Follow the Post-keyboarding Workflow for Books, starting with inserting full TEI headers and continuing through finalization.
    TIP

    After inserting the full TEI headers, the QA program will complain that "the TEI header should not contain any '&resp_transcription...;' or '&resp_markup...;' or '&resp_ocr...;' entity references." This message is correct; for non-transcription TEI files it doesn't make sense to specify who was responsible for the transcription or the markup. Search the TEI files for the kinds of entity references specified in the error message, and remove them.
